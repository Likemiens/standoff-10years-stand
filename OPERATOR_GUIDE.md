# Инструкция стенда Standoff

Эта инструкция нужна для запуска стенда после включения компьютера и для базовой проверки на площадке.

## 1. Перед запуском

Проверьте:

- ноутбук подключен к питанию;
- основной экран подключен к ноутбуку;
- обе Arduino подключены в USB-порты;
- на рабочем столе есть ярлык `Standoff`;
- звук не важен: приложение запускает ролики без звука.

Важно: на рабочем столе должен быть именно ярлык, а не скопированный файл `START_STANDOFF.bat`.

## 2. Обычный запуск

1. Включите ноутбук.
2. Дождитесь загрузки Windows.
3. Дважды нажмите ярлык `Standoff` на рабочем столе.
4. Приложение откроется на весь экран.
5. На экране должна появиться заставка стенда.

Если стенд настроен на автозапуск, после входа в Windows приложение откроется само.

## 3. Как работает стенд

Посетитель вращает два колеса с цифрами.

Приложение собирает год:

```text
десятки + единицы = YY
полный год = 20YY
```

Примеры:

```text
1 и 7 -> 17 -> 2017
2 и 4 -> 24 -> 2024
0 и 9 -> 09 -> 2009
```

Логика показа:

```text
00-15 -> ролик/экран before_2016
16-26 -> ролик конкретного года 2016-2026
27-99 -> ролик/экран after_2026
```

После окончания ролика приложение само возвращается в заставку.

Важно: значение, которое уже стояло на колесах в момент запуска приложения, может не запустить ролик сразу. Это защита от случайного автозапуска. Чтобы проверить запуск, измените цифры на колесах.

## 4. Проверка, что стенд работает

После запуска:

1. На экране видна заставка.
2. Поверните колеса на год из диапазона `2016-2026`, например `2017`.
3. Подождите около 1 секунды.
4. Должен запуститься ролик выбранного года.
5. После окончания ролика должна вернуться заставка.

## 5. Debug-панель

Если нужно проверить состояние стенда, нажмите:

```text
F1
```

В debug-панели важны поля:

```text
Arduino status       -> connected
COM-port             -> должны быть COM-порты двух Arduino
Last raw input       -> последнее считанное значение
Stable value         -> собранное YY
Mapped year          -> выбранный год
Current scenario     -> текущий сценарий
Current file         -> текущий файл контента
Last error           -> последняя ошибка
```

Чтобы закрыть debug-панель, снова нажмите `F1`.

## 6. Ручной режим

Если Arduino не работает или нужно вручную проверить контент, нажмите:

```text
F2
```

В ручном режиме можно:

- ввести `YY`, например `17`;
- запустить конкретный год `2016-2026`;
- запустить `Idle`;
- запустить `Before 2016`;
- запустить `After 2026`.

Чтобы закрыть ручной режим, снова нажмите `F2`.

## 7. Если что-то пошло не так

### На экране нет приложения

1. Запустите ярлык `Standoff` на рабочем столе.
2. Если не помогло, откройте папку:

```text
C:\Standoff\StandoffHistory\
```

3. Запустите:

```text
START_STANDOFF.bat
```

Если нужен ярлык на рабочем столе, запустите из этой же папки:

```text
CREATE_DESKTOP_SHORTCUT.bat
```

Не копируйте `START_STANDOFF.bat` на рабочий стол вручную.

### Приложение открыто, но колеса не запускают ролики

1. Нажмите `F1`.
2. Проверьте `Arduino status`.
3. Если там `disconnected`, переподключите USB Arduino.
4. Закройте приложение и запустите его снова.

### Ролик не найден

1. Нажмите `F1`.
2. Посмотрите `Last error` и `Current file`.
3. Проверьте, что нужный файл лежит в папке:

```text
C:\Standoff\StandoffHistory\content\
```

Имена файлов должны быть:

```text
standoff_idle.mp4
standoff_before_2016.mp4
standoff_after_2026.mp4
standoff_2016.mp4
...
standoff_2026.mp4
```

### Нужно быстро перезапустить стенд

1. Закройте приложение через `Alt + F4`.
2. Запустите ярлык `Standoff` заново.

Если приложение не закрывается, нажмите `Ctrl + Shift + Esc`, откройте Диспетчер задач и завершите `StandoffHistory.exe`.

## 8. Что нельзя делать во время работы

- Не запускать две копии приложения одновременно.
- Не запускать `serial_check.py`, пока открыто основное приложение.
- Не вытаскивать USB Arduino во время показа.
- Не переименовывать файлы в `content`.
- Не удалять `config.json`.

## 9. Где лежат логи

Логи находятся здесь:

```text
C:\Standoff\StandoffHistory\logs\events.log
```

Этот файл нужен техническому специалисту, если стенд работает нестабильно.

## 10. Выключение стенда

1. Закройте приложение через `Alt + F4`.
2. Завершите работу Windows обычным способом.
3. После выключения ноутбука можно отключать питание стенда.
4. Отключать USB от ноутбука нежелательно, они отвечают за считывание цифр. Смена портов между другогом может изменить ввод данных на экране.
