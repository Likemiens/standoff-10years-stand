# Инструкция оператора стенда Standoff

Эта инструкция описывает обычный запуск стенда на площадке и базовую проверку, что экран, колеса и подсветка работают автономно.

## 1. Что подключено

На стенде используется:

- ноутбук Windows;
- внешний экран по HDMI;
- две платы для считывания колес;
- отдельная плата подсветки;
- файл/ярлык `standoff` на рабочем столе для запуска приложения.

Рабочая схема портов:

```text
COM3 -> третья цифра года, десятки YY
COM4 -> четвертая цифра года, единицы YY
COM5 -> подсветка
```

Пример:

```text
COM3 прислал 1
COM4 прислал 7
приложение собрало 17 -> 2017
```

## 2. Обычный запуск

1. Включить ноутбук.
2. Дождаться загрузки Windows.
3. Убедиться, что внешний экран включен и подключен по HDMI.
4. Дважды нажать файл/ярлык `standoff` на рабочем столе.
5. Приложение должно открыться на большом экране в fullscreen.
6. На экране должна появиться заставка.
7. Подсветка в заставке должна быть выключена: команда `0`.

Если приложение открылось на экране ноутбука, а не на внешнем экране, нужен технический специалист: в `config.json` нужно проверить `display.screenIndex`.

## 3. Как работает сценарий

Посетитель вращает два колеса.

Приложение собирает год так:

```text
YY = третья цифра + четвертая цифра
полный год = 20YY
```

Примеры:

```text
1 + 7 -> 17 -> 2017
2 + 4 -> 24 -> 2024
0 + 9 -> 09 -> 2009
```

Логика контента:

```text
00-15 -> before_2016
16-26 -> ролик конкретного года 2016-2026
27-99 -> after_2026
```

После окончания ролика приложение само возвращается в заставку.

Важно: значение, которое уже стояло на колесах при запуске приложения, может не запустить ролик сразу. Это защита от случайного автозапуска. Для проверки нужно изменить цифры на колесах.

## 4. Как работает подсветка

Подсветка управляется отдельной платой на `COM5`.

Команды подсветки:

```text
2 -> анимация от барабана к экрану при выборе правильного года
1 -> кратко зажечь линии при выборе года вне диапазона
0 -> выключить
```

Автоматический сценарий:

```text
приложение запустилось -> 0
стенд в заставке -> 0
выбран год 2016-2026 -> 2
выбран год 00-15 или 27-99 -> 1
через 1.2 секунды валидный ролик запускается, подсветка остается в режиме 2
через 7 секунд выбор вне диапазона гаснет -> 0
валидный ролик закончился -> 0
возврат в заставку -> 0
```

Если выбран год вне диапазона `2016-2026`, приложение отправит команду `1`, покажет `before_2016` или `after_2026`, затем отправит `0` через `outOfRangeDurationMs`.

## 5. Быстрая проверка перед открытием зоны

1. Запустить `standoff` с рабочего стола.
2. Проверить, что на большом экране видна заставка.
3. Проверить, что подсветка выключена.
4. Повернуть колеса на год `2017`.
5. Должен запуститься ролик `2017`.
6. Подсветка должна запустить анимацию от барабана к экрану и работать до окончания ролика.
7. После окончания ролика подсветка должна выключиться и должна вернуться заставка.
8. Подсветка должна остаться выключенной.

## 6. Debug-панель

Для проверки состояния нажать:

```text
F1
```

В debug-панели нормальное состояние:

```text
Arduino status       -> connected
COM-port             -> tens=COM3:connected; ones=COM4:connected
Current scenario     -> idle или текущий сценарий
Current file         -> текущий видеофайл
LED status           -> ok
Last LED command     -> 0, 1 или 2
Last error           -> -
Content check status -> ok
```

Важно: `COM5` может не отображаться в поле `COM-port`, потому что это отдельный порт подсветки, а поле `COM-port` показывает платы колес.

Закрыть debug-панель: снова нажать `F1`.

## 7. Ручной режим

Для ручной проверки нажать:

```text
F2
```

В ручном режиме можно:

- ввести `YY`, например `17`;
- запустить конкретный ролик `2016-2026`;
- запустить `Idle`;
- запустить `Before 2016`;
- запустить `After 2026`;
- проверить подсветку кнопками `RUN`, `OFF`, `IDLE`, `ERROR`.

Соответствие кнопок подсветки:

```text
RUN   -> 2
OFF   -> 0
IDLE  -> 0
ERROR -> 0
```

Закрыть ручной режим: снова нажать `F2`.

## 8. Если приложение не запускается

1. Еще раз запустить `standoff` на рабочем столе.
2. Если не помогло, открыть папку приложения:

```text
C:\Standoff\StandoffHistory\
```

3. Запустить:

```text
START_STANDOFF.bat
```

Важно: на рабочем столе должен быть ярлык или корректный файл запуска `standoff`. Не нужно вручную копировать `START_STANDOFF.bat` на рабочий стол, если после этого он не видит `StandoffHistory.exe`.

## 9. Если колеса не запускают ролик

1. Нажать `F1`.
2. Проверить поле `Arduino status`.
3. Проверить поле `COM-port`.

Нормально:

```text
tens=COM3:connected; ones=COM4:connected
```

Если там другие порты или ошибка:

- закрыть приложение;
- проверить USB-подключение плат;
- запустить приложение снова;
- если не помогло, позвать технического специалиста.

## 10. Если подсветка не работает

1. Нажать `F1`.
2. Проверить:

```text
LED status
Last LED command
Last error
```

Если `Last LED command` меняется на `2`, `1` или `0`, но физически ничего не происходит, значит приложение команды отправляет, а проблему нужно искать в плате подсветки, питании или проводке.

Если в ошибке есть `COM5 PermissionError`, значит `COM5` занят другой программой. Нужно закрыть все тестовые скрипты и вторые копии приложения.

## 11. Если ролик не найден

1. Нажать `F1`.
2. Посмотреть `Last error` и `Current file`.
3. Проверить папку:

```text
C:\Standoff\StandoffHistory\content\
```

В ней должны быть файлы:

```text
standoff_idle.mp4
standoff_before_2016.mp4
standoff_after_2026.mp4
standoff_2016.mp4
...
standoff_2026.mp4
```

## 12. Что нельзя делать во время работы

- Не запускать две копии приложения одновременно.
- Не запускать `serial_check.py` или `led_check.py`, пока открыто основное приложение.
- Не вытаскивать USB-платы во время работы.
- Не менять USB-порты местами без технического специалиста.
- Не переименовывать файлы в `content`.
- Не удалять `config.json`.

## 13. Перезапуск

Если нужен быстрый перезапуск:

1. Закрыть приложение через `Alt + F4`.
2. Запустить `standoff` на рабочем столе снова.

Если приложение не закрывается:

1. Нажать `Ctrl + Shift + Esc`.
2. Открыть Диспетчер задач.
3. Завершить `StandoffHistory.exe`.
4. Запустить `standoff` снова.

## 14. Выключение стенда

1. Закрыть приложение через `Alt + F4`.
2. Завершить работу Windows обычным способом.
3. После выключения ноутбука можно отключать питание стенда.

## 15. Техническая заметка

Рабочий `config.json` для стенда должен содержать:

```json
"serial": {
  "enabled": true,
  "mode": "dual_digit",
  "port": "auto",
  "baudRate": 9600,
  "lineEnding": "\n",
  "reconnectIntervalMs": 2000,
  "dual": {
    "tensPort": "COM3",
    "onesPort": "COM4",
    "ledTarget": "tens"
  }
},
"led": {
  "enabled": true,
  "mode": "serial",
  "port": "COM5",
  "baudRate": 9600,
  "lineEnding": "\n",
  "openDelayMs": 1500,
  "runCommand": "2",
  "outOfRangeCommand": "1",
  "offCommand": "0",
  "idleCommand": "0",
  "errorCommand": "0",
  "runDelayMs": 1200,
  "outOfRangeDurationMs": 7000
}
```

Если приложение запускается из собранной папки, править нужно:

```text
C:\Standoff\StandoffHistory\config.json
```

Если приложение запускается из репозитория через Python, править нужно:

```text
C:\Standoff\standoff-10years-stand\config.json
```
