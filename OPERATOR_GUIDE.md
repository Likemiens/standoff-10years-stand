# Инструкция оператора Standoff

Короткая инструкция для запуска стенда, проверки работы и решения частых проблем на площадке.

## 1. Как включить стенд

1. Включите ноутбук.
2. Дождитесь загрузки Windows.
3. Проверьте, что большой экран включен и подключен по HDMI.
4. На рабочем столе дважды нажмите `standoff`.
5. Приложение должно открыться на большом экране во весь экран.
6. На экране должна появиться заставка.

В заставке подсветка должна быть выключена.

## 2. Быстрая проверка

После запуска проверьте:

1. На большом экране видна заставка.
2. Поверните колеса на год `2017`.
3. Подсветка должна запустить анимацию от барабана к экрану.
4. Должен запуститься ролик `2017`.
5. После ролика приложение должно вернуться в заставку.
6. Подсветка снова должна выключиться.

Проверьте неправильный год:

1. Поверните колеса на `2009` или `2030`.
2. Подсветка должна кратко загореться.
3. Должен запуститься экран/ролик `before_2016` или `after_2026`.
4. После этого стенд должен вернуться в заставку.

## 3. Как пользоваться

Посетитель вращает два колеса с цифрами.

Приложение собирает год:

```text
1 и 7 -> 2017
2 и 4 -> 2024
0 и 9 -> 2009
```

Логика:

```text
2000-2015 -> before_2016
2016-2026 -> ролик выбранного года
2027-2099 -> after_2026
```

Первое значение после запуска может не включить ролик сразу. Это нормально: защита от случайного запуска. Нужно изменить цифры на колесах.

## 4. Debug-панель

Если нужно понять, что происходит, нажмите:

```text
F1
```

Нормальное состояние:

```text
Arduino status       -> connected
COM-port             -> tens=COM3:connected; ones=COM4:connected
LED status           -> ok
Last LED command     -> 0, 1 или 2
Content check status -> ok
Last error           -> -
```

Команды подсветки:

```text
0 -> выключена
1 -> кратко светится для года вне диапазона
2 -> анимация от барабана к экрану
```

Чтобы закрыть debug-панель, снова нажмите `F1`.

## 5. Ручной режим

Если колеса не работают или нужно вручную проверить ролики, нажмите:

```text
F2
```

В ручном режиме можно:

- ввести `YY`, например `17`;
- запустить ролики `2016-2026`;
- запустить `Idle`;
- запустить `Before 2016`;
- запустить `After 2026`;
- проверить подсветку кнопками `RUN`, `OFF`, `IDLE`, `ERROR`.

Кнопки подсветки:

```text
RUN  -> 2
OFF  -> 0
IDLE -> 0
ERROR -> 0
```

Чтобы закрыть ручной режим, снова нажмите `F2`.

## 6. Если приложение не запустилось

1. Еще раз нажмите `standoff` на рабочем столе.
2. Если не помогло, откройте папку:

```text
C:\Standoff\StandoffHistory\
```

3. Запустите:

```text
START_STANDOFF.bat
```

Если появляется ошибка, что `StandoffHistory.exe` не найден, значит на рабочий стол был скопирован неправильный файл запуска. Нужен ярлык или файл `standoff`, который запускает программу из папки приложения.

## 7. Если приложение открылось не на большом экране

1. Закройте приложение через `Alt + F4`.
2. Проверьте, что HDMI-экран включен.
3. Нажмите `Win + P`.
4. Выберите режим `Расширить`.
5. Запустите `standoff` снова.

Если приложение снова открылось на экране ноутбука, нужен технический специалист.

## 8. Если колеса не запускают ролик

1. Нажмите `F1`.
2. Проверьте:

```text
Arduino status -> connected
COM-port       -> tens=COM3:connected; ones=COM4:connected
```

Если статус не `connected`:

1. Закройте приложение через `Alt + F4`.
2. Проверьте USB-подключение плат.
3. Запустите `standoff` снова.

Если не помогло, нужен технический специалист.

## 9. Если подсветка не работает

1. Нажмите `F1`.
2. Проверьте:

```text
LED status
Last LED command
Last error
```

Если `Last LED command` меняется на `0`, `1` или `2`, но подсветка не реагирует, приложение команды отправляет. Нужно проверить питание, плату подсветки или проводку.

Если в ошибке есть `COM5 PermissionError`, значит порт подсветки занят другой программой. Закройте все вторые копии приложения и тестовые скрипты.

## 10. Если ролик не найден

1. Нажмите `F1`.
2. Посмотрите `Last error` и `Current file`.
3. Проверьте, что видео лежат в папке:

```text
C:\Standoff\StandoffHistory\content\
```

Нужные файлы:

```text
standoff_idle.mp4
standoff_before_2016.mp4
standoff_after_2026.mp4
standoff_2016.mp4
...
standoff_2026.mp4
```

## 11. Что нельзя делать

- Не запускать две копии приложения одновременно.
- Не запускать `serial_check.py` или `led_check.py`, пока открыто основное приложение.
- Не вытаскивать USB-платы во время работы.
- Не менять USB-порты местами.
- Не переименовывать файлы в `content`.
- Не удалять `config.json`.

## 12. Как выключить стенд

1. Закройте приложение через `Alt + F4`.
2. Завершите работу Windows обычным способом.
3. После выключения ноутбука можно отключать питание стенда.
