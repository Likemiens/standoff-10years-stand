# План тестирования Standoff History App

## 1. Подготовка окружения

Цель: убедиться, что приложение запускается на целевой машине.

1. Проверить версию Python:

```bash
python --version
```

Ожидаемо: Python 3.11 или новее.

2. Создать и активировать виртуальное окружение:

```bash
python -m venv .venv
.venv\Scripts\activate
```

3. Установить зависимости:

```bash
pip install -r requirements.txt
```

4. Проверить наличие основных файлов:

```text
main.py
config.json
requirements.txt
app/
tools/
content/
logs/
```

Ожидаемо: структура проекта на месте, ошибок установки нет.

## 2. Проверка контента

Цель: убедиться, что все нужные ролики и заглушки лежат в `content/`.

Обязательные файлы:

```text
standoff_2016.mp4
standoff_2017.mp4
standoff_2018.mp4
standoff_2019.mp4
standoff_2020.mp4
standoff_2021.mp4
standoff_2022.mp4
standoff_2023.mp4
standoff_2024.mp4
standoff_2025.mp4
standoff_2026.mp4
standoff_idle.mp4
standoff_before_2016.mp4
standoff_after_2026.mp4
```

Для `idle`, `before_2016`, `after_2026` допустимы также `.png`, `.jpg`, `.jpeg`.

Запуск проверки:

```bash
python tools/content_check.py
```

Ожидаемо:

```text
Status: OK
```

Если есть `missing`, `empty` или `unreadable`, нужно заменить или добавить указанные файлы.

## 3. Тест без Arduino

Цель: проверить GUI, fullscreen, ручной режим, debug и маппинг годов без железа.

Запуск:

```bash
python main.py --simulate
```

Проверить:

- приложение открылось;
- показывается idle;
- окно fullscreen, если в `config.json` стоит `"fullscreen": true`;
- курсор скрыт, если стоит `"hideCursor": true`;
- `F1` открывает и закрывает debug-панель;
- `F2` открывает и закрывает ручной режим.

Быстрые клавиши simulate:

```text
I -> idle
B -> before_2016
A -> after_2026
V -> 2017
```

Ожидаемо: каждый сценарий запускает соответствующий контент или понятную техническую ошибку, если файла нет.

## 4. Проверка ручного режима

Цель: убедиться, что зона может работать без Arduino.

Открыть ручной режим:

```text
F2
```

Проверить ввод `YY`:

| Ввод | Ожидаемый сценарий |
|---|---|
| `00` | `before_2016` |
| `01` | `before_2016` |
| `15` | `before_2016` |
| `16` | `2016.mp4` |
| `17` | `2017.mp4` |
| `26` | `2026.mp4` |
| `27` | `after_2026` |
| `42` | `after_2026` |
| `99` | `after_2026` |

Проверить кнопки:

- `Idle`;
- `Before 2016`;
- `After 2026`;
- ролики `2016-2026`;
- `LED_RUN`;
- `LED_OFF`;
- `LED_IDLE`;
- `LED_ERROR`.

Ожидаемо: ручной запуск работает даже без подключенной Arduino.

## 5. Проверка debug-панели

Цель: убедиться, что монтажник видит состояние системы.

Открыть:

```text
F1
```

Проверить поля:

- `Arduino status`;
- `COM-port`;
- `Last raw input`;
- `Candidate value`;
- `Stable value`;
- `Last triggered value`;
- `Mapped year`;
- `Current scenario`;
- `Current file`;
- `LED status`;
- `Last LED command`;
- `Last error`;
- `Content check status`.

Ожидаемо: значения обновляются при ручном запуске, simulate-командах и Serial-вводе.

## 6. Проверка Arduino / Serial

Цель: убедиться, что Arduino отправляет корректный формат `YY\n`.

Подключить Arduino по USB.

Проверить список портов:

```bash
python tools/serial_check.py --port auto --baud 9600
```

Если auto не нашёл порт, указать порт вручную:

```bash
python tools/serial_check.py --port COM3 --baud 9600
```

Ожидаемо:

```text
valid: '17'
valid: '05'
valid: '42'
```

Некорректные значения должны отображаться как `invalid`:

```text
invalid: '2017'
invalid: '7'
invalid: 'A1'
```

## 7. Основной тест с Arduino

Цель: проверить полный сценарий барабан → Arduino → приложение → экран.

Запуск:

```bash
python main.py
```

Проверить:

1. На старте показывается idle.
2. Если Arduino не найдена, приложение не падает.
3. В debug видно `Arduino status`.
4. При изменении барабана приложение получает `YY`.
5. Значение стабилизируется через `stabilizationDelayMs`.
6. Полный год собирается как `20YY`.
7. Запускается правильный сценарий.
8. После завершения ролика приложение возвращается в idle.

## 9. Проверка защиты от повторов

Цель: убедиться, что один и тот же год не перезапускается бесконечно.

Шаги:

1. Отправить `17`.
2. Дождаться запуска `2017.mp4`.
3. Повторно отправить `17` несколько раз.

Ожидаемо:

- ролик не перезапускается бесконечно;
- в логе есть `duplicate_ignored`;
- если затем отправить `18`, текущий контент меняется на `2018.mp4`.

## 10. Проверка ошибок

Цель: убедиться, что ошибки не ломают основной экран.

Проверить:

| Ситуация | Ожидаемое поведение |
|---|---|
| Arduino не подключена | idle, debug показывает disconnected, ручной режим работает |
| Arduino отключили во время работы | текущий ролик не падает, приложение пробует reconnect |
| Пришло `2017` вместо `17` | значение игнорируется, экран не меняется |
| Файл ролика отсутствует | техническая заглушка, ошибка в debug и логе |
| `idle` отсутствует | техническая заглушка, приложение не падает |

## 11. Проверка LED

Цель: проверить LED-команды без блокировки основного сценария.

Если LED пока не подключена, оставить:

```json
"led": {
  "enabled": false
}
```

Ожидаемо: приложение работает без ленты.

Если LED подключена через Arduino/Serial, включить:

```json
"led": {
  "enabled": true,
  "mode": "serial",
  "runCommand": "LED_RUN",
  "offCommand": "LED_OFF",
  "idleCommand": "LED_IDLE",
  "errorCommand": "LED_ERROR"
}
```

Проверить из ручного режима:

- `LED_RUN`;
- `LED_OFF`;
- `LED_IDLE`;
- `LED_ERROR`.

Проверить автоматический сценарий:

1. Отправить год `17`.
2. Ожидаемо: отправляется `LED_RUN`.
3. Через `runDelayMs` запускается ролик.
4. Если `sendOffOnVideoStart=true`, отправляется `LED_OFF`.

## 12. Проверка логов

Лог находится здесь:

```text
logs/events.log
```

Проверить, что логируются:

- запуск приложения;
- загрузка сценариев;
- подключение или отсутствие Arduino;
- валидные входящие значения;
- невалидные входящие значения;
- stable value;
- выбранный год;
- текущий файл;
- ручные запуски;
- LED-команды;
- ошибки контента;
- возврат в idle.

## 13. Финальный прогон перед площадкой

Чеклист:

- [ ] Python 3.11+ установлен.
- [ ] Зависимости установлены.
- [ ] `python tools/content_check.py` возвращает `Status: OK`.
- [ ] `python main.py --simulate` запускается.
- [ ] `F1` debug работает.
- [ ] `F2` manual работает.
- [ ] Все сценарии `00-99` критичного набора проверены.
- [ ] Arduino определяется.
- [ ] `serial_check.py` показывает `valid` значения.
- [ ] `python main.py` работает с Arduino.
- [ ] Повтор одного значения не перезапускает ролик.
- [ ] Новый год во время ролика переключает сценарий.
- [ ] После завершения ролика происходит возврат в idle.
- [ ] Ошибки контента не роняют приложение.
- [ ] LED-команды проверены или `led.enabled=false`.
- [ ] Логи пишутся в `logs/events.log`.
- [ ] На целевом ПК отключены сон, заставка, уведомления и автоотключение экрана.
